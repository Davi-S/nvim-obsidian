*nvim-obsidian.txt*  nvim-obsidian

nvim-obsidian is a configurable Obsidian workflow plugin for Neovim.

COMMANDS
  :ObsidianOmni              Open omni picker (search/create).
  :ObsidianToday             Open or create today's daily note.
  :ObsidianNext              Open or create next journal note for current context.
  :ObsidianPrev              Open or create previous journal note for current context.
  :ObsidianFollow            Follow wiki-link under cursor.
  :ObsidianBacklinks         Search backlinks for current note title.
  :ObsidianSearch            Vault-scoped text search (Telescope live_grep with vault_root cwd).
  :ObsidianReindex           Force full vault cache rebuild.
  :ObsidianInsertTemplate [type|path]
                             Insert rendered template at cursor.
                             Arg can be a note type (standard/daily/weekly/monthly/yearly)
                             or a file path to a template file.

CONFIG
  setup({
    vault_root = "/abs/path",
    locale = "pt-BR",
    new_notes_subdir = "10 Novas notas",

DATAVIEW CONFIG
  First-wave dataview options are available under the dataview setup key.

  Shape:
    dataview = {
      enabled = true,
      render = {
        when = { "on_open", "on_save" },
        scope = "event",       -- event | current | visible | loaded
        patterns = { "*.md" },
      },
      placement = "below_block", -- below_block | above_block
      messages = {
        task_no_results = {
          enabled = true,
          text = "Dataview: No results to show for task query.",
        },
      },
      highlights = {
        -- optional highlight group overrides
        -- header = "markdownLinkText",
        -- error = "WarningMsg",
        -- table_link = "markdownLinkText",
        -- task_no_results = "Comment",
      },
    }

  render.when options (multi-select):
    on_open      -> BufReadPost
    on_save      -> BufWritePost
    on_buf_enter -> BufEnter

  render.scope options:
    event   = refresh only the event buffer
    current = refresh only current buffer
    visible = refresh visible-window markdown buffers
    loaded  = refresh all loaded markdown buffers

  The "cmd" option in lazy.nvim is NOT a plugin config field - it tells lazy.nvim
  when to load the plugin. If any of the listed commands are invoked, the plugin
  is loaded automatically. This avoids loading the plugin until actually needed.

  Example lazy.nvim snippet (NOT part of plugin config):
    cmd = { "ObsidianOmni", "ObsidianToday", "ObsidianNext", ... }

API QUICK REFERENCE
  local obsidian = require("nvim-obsidian")

  -- Template engine (note body templates)
  obsidian.template_register_placeholder("title", function(ctx)
    return ctx.note.title
  end)

  -- Journal title formats (render + parse)
  obsidian.journal.register_placeholder("year", function(ctx)
    return tostring(ctx.date.year)
  end, "(%d%d%d%d)")

JOURNAL CONFIG
  Journal notes are organized in a nested configuration where each note type
  (daily, weekly, monthly, yearly) contains its own subdir, title_format, and
  optional template.
  
  Structure:
    journal = {
      daily = {
        subdir = "path/to/daily",
        title_format = "{{year}} {{month_name}}",
        template = "path/to/template",  -- optional
      },
      weekly = { subdir = "...", title_format = "..." },
      monthly = { subdir = "...", title_format = "..." },
      yearly = { subdir = "...", title_format = "..." },
    }

  Journal placeholders must be explicitly registered before setup.
  API:
    require("nvim-obsidian").journal.register_placeholder(name, resolver, regex_fragment)

  Default-compatible registrations:
    local obsidian = require("nvim-obsidian")
    obsidian.journal.register_placeholder("year", function(ctx)
      return tostring(ctx.date.year)
    end, "(%d%d%d%d)")

    obsidian.journal.register_placeholder("iso_year", function(ctx)
      return tostring(ctx.date.iso_year)
    end, "(%d%d%d%d)")

    obsidian.journal.register_placeholder("month_name", function(ctx)
      return ctx.locale.month_name or ""
    end, "(.+)")

    obsidian.journal.register_placeholder("day2", function(ctx)
      return string.format("%02d", ctx.date.day or 0)
    end, "(%d%d?)")

    obsidian.journal.register_placeholder("weekday_name", function(ctx)
      return ctx.locale.weekday_name or ""
    end, "(.+)")

    obsidian.journal.register_placeholder("iso_week", function(ctx)
      return tostring(ctx.date.iso_week)
    end, "(%d%d?)")
  
  Example:
    journal = {
      daily = {
        subdir = "11 Diário/11.01 Diário",
        title_format = "{{year}} {{month_name}} {{day2}}, {{weekday_name}}",
        template = "08 Templates/Nota diária",
      },
      weekly = {
        subdir = "11 Diário/11.02 Semanal",
        title_format = "{{iso_year}} semana {{iso_week}}",
      },
      monthly = {
        subdir = "11 Diário/11.03 Mensal",
        title_format = "{{year}} {{month_name}}",
      },
      yearly = {
        subdir = "11 Diário/11.04 Anual",
        title_format = "{{year}}",
      },
    }

TEMPLATES
  The plugin provides a flexible template system with user-configurable 
  placeholders. All placeholders must be registered via template_register_placeholder().

  Placeholder syntax:
    {{placeholder_name}}  - replaced with resolver output
    {{unknown}}           - kept as-is if not registered (with warning)

  Registration (call during setup):
    require("nvim-obsidian").template_register_placeholder("title", function(ctx)
      return ctx.note.title
    end)

    require("nvim-obsidian").template_register_placeholder("date", function(ctx)
      return ctx.time.format_local("%Y-%m-%d")
    end)

  Context object fields:
    ctx.note.title        - note title string
    ctx.note.type         - note type (standard/daily/weekly/monthly/yearly)
    ctx.note.input        - original user input
    ctx.note.rel_path     - vault-relative path
    ctx.note.aliases      - table of aliases
    ctx.note.tags         - table of tags
    ctx.note.abs_path     - absolute file path
    
    ctx.time.timestamp    - unix timestamp
    ctx.time.local        - local date table {year,month,day,hour,min,sec,wday,yday}
    ctx.time.utc          - UTC date table
    ctx.time.iso          - ISO object {date, datetime, week, year}
    ctx.time.format_local(fmt)  - format function, e.g. "%Y-%m-%d"
    ctx.time.format_utc(fmt)    - UTC format function, e.g. "%Y-%m-%d"
    
    ctx.config            - read-only config object

  Template usage:
    :ObsidianInsertTemplate              auto-detect note type
    :ObsidianInsertTemplate standard     use 'standard' note template
    :ObsidianInsertTemplate ./path.md    load template from file

  Common placeholders (examples):
    template_register_placeholder("title", function(ctx) return ctx.note.title end)
    template_register_placeholder("date", function(ctx) return ctx.time.format_local("%Y-%m-%d") end)
    template_register_placeholder("time", function(ctx) return ctx.time.format_local("%H:%M") end)
    template_register_placeholder("weekday", function(ctx) return ctx.time.format_local("%A") end)
    template_register_placeholder("year", function(ctx) return tostring(ctx.time.iso.year) end)
    template_register_placeholder("note_type", function(ctx) return ctx.note.type end)

SCOPE
  No visual concealment.
  No markdown formatting.

OMNI POLICY
  Search objects are ordered as: title, aliases, relpath.
  Display defaults to: title -> relpath.
  If query matches an alias and not title, display becomes: alias -> relpath.

SYNC
  Cache sync is driven by write/buffer events, focus-gained reconciliation,
  and best-effort filesystem watchers for external vault changes.
